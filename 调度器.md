# 容忍与污点
相互配合，用于避免pod被分配到不合适的节点上
每个节点可以应用一个或多个污点，不能容忍这些污点的pod不会被该节点接收

``key=value:effect`` value可空

```bash
kubectl taint node k8s-master01 computeengine=gpu:NoSchedule
# 主节点设置污点，除非能容忍computeengine=gpu，否则禁止调度

kubectl taint node k8s-master01 computeengine=gpu:NoSchedule-
# 加个-，移除污点
```

effect：
	NoSchedule
	PreferNoSchedule
	NoExecute

master节点自带污点：不参与调度
