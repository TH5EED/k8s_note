# 容器介绍
容器技术的兴起源于 PaaS 技术的普及；
Docker 公司发布的 Docker 项目具有里程碑式的意义；
Docker 项目通过“容器镜像”，解决了应用打包这个根本性难题。

---

程序是代码的二进制文件+需要用到的数据文件
当程序一旦被执行起来，需要用到各种计算机执行环境（比如寄存器，内存堆栈等），就叫做进程

对于进程来说
	静态表现就是程序
	动态表现就是计算机的数据和状态的总和

容器核心技术在于==约束和修改进程的动态表现==，为容器创造一个边界
主要技术为Cgroups制造约束，Namespace修改进程视图
## Namespace技术

Namespace是Linux创建新进程的一个可选参数CLONE_NEWPID
新进程会”看到“一个新的进程空间（PID Namespace），此进程会认为自己是第1号进程
利用==障眼法==实现了修改进程视图，实现了最基本的隔离效果

==容器在Linux内核视角来看，就是一个特殊隔离属性的进程==

## Cgroups技术
多个容器之间虽然相互隔离，但并非绝对隔离
- 共同使用宿主机的系统内核
- Linux内核中有许多资源和对象不能被Namespace化，比如时间

==Cgroups技术用来为进程设置资源限制实现制造约束的效果==
